## 2. 推荐总体顺序

1. 先做 P0，把“新装实例公告空白”问题彻底打掉。
2. 再做 P1，把“检查更新”和“同步公告”收口到同一个入口。
3. 随后做 P2，把单机更新的 preflight、日志、核验闭环做实。
4. 再推进 P3，把集群排空和批次可视化补齐。
5. 最后用 P4 收口回滚、审计、策略化，不建议提前做。
